# OneTune — Deploy on Render + MongoDB Atlas

> **Step-by-step production guide:** MongoDB Atlas → Render backend services → **local Android APK** → install with ADB.

Use this when you want **managed cloud** instead of a VPS. Each backend service runs as its own Render web service; the database lives on **MongoDB Atlas**.

**Local dev** stays the same: `docker compose up -d` — see [DEVELOPMENT.md](./DEVELOPMENT.md).

---

## What you will deploy

| Component | Where | Public URL? |
|-----------|--------|-------------|
| MongoDB | MongoDB Atlas | No (connection string only) |
| Extractor (yt-dlp) | Render | Yes (`*.onrender.com`) |
| Spotify scraper | Render | Yes |
| JioSaavn API | Render | Yes |
| **API (main)** | Render | **Yes — mobile points here** |
| Mobile app | Local APK build → ADB install | Talks to API over HTTPS |

```
  Phone (release APK)
        │
        ▼
  https://OneTune-api.onrender.com
        │
        ├──► MongoDB Atlas (mongodb+srv://…)
        ├──► OneTune-extractor.onrender.com
        ├──► OneTune-spotify.onrender.com
        └──► OneTune-jiosaavn.onrender.com
```

## Part 1 — MongoDB Atlas

### 1.1 Create a cluster

1. Log in to [MongoDB Atlas](https://cloud.mongodb.com).
2. **Create** → **Shared (M0)** → pick a cloud region **close to your Render region** (e.g. both `us-east`).
3. Name the cluster (e.g. `OneTune`).

### 1.2 Database user

1. **Database Access** → **Add New Database User**.
2. Authentication: **Password**.
3. Username: `OneTune` (or any name).
4. Password: generate a strong password — **save it**.
5. Privileges: **Read and write to any database** (or restrict to `OneTune` DB).

### 1.3 Network access

Render uses dynamic IPs. For Atlas:

1. **Network Access** → **Add IP Address**.
2. Choose **Allow Access from Anywhere** (`0.0.0.0/0`).

> Tighter security: use Atlas VPC peering or Render static egress on paid plans — optional for personal/household deploys.

### 1.4 Connection string

1. **Database** → **Connect** → **Drivers**.
2. Copy the connection string, e.g.:

```
mongodb+srv://OneTune:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
```

3. Edit it:

- Replace `<password>` with your DB user password (URL-encode special chars).
- Add database name before `?`:

```
mongodb+srv://OneTune:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/OneTune?retryWrites=true&w=majority
```

Save this as **`MONGODB_URI`** — you will paste it into Render in Part 3.

### 1.5 Verify (optional)

From your PC with [MongoDB Compass](https://www.mongodb.com/products/compass) or `mongosh`, connect with the same URI.

## Part 3 — Deploy backend on Render

Two options: **Blueprint (recommended)** or **manual**.

### Option A — Blueprint (one click, 5 services)

1. [Render Dashboard](https://dashboard.render.com) → **New** → **Blueprint**.
2. Connect your **OneTune** GitHub repo.
3. Render detects `render.yaml` at the repo root.
4. Review services:
   - `OneTune-extractor`
   - `OneTune-spotify`
   - `OneTune-jiosaavn`
   - `OneTune-api`
5. When prompted for **`MONGODB_URI`** on `OneTune-api`, paste your Atlas connection string from Part 1.
6. **`JWT_SECRET`** is auto-generated by the blueprint.
7. Click **Apply**.

**First deploy:** JioSaavn image build can take **10–15 minutes**. Wait until all services show **Live**.

### Option B — Manual services

Create four **Web Services** → **Docker**:

| Service name | Dockerfile | Health check |
|--------------|------------|--------------|
| `OneTune-extractor` | `docker/extractor.Dockerfile` | `/health` |
| `OneTune-spotify` | `docker/spotify.Dockerfile` | `/health` |
| `OneTune-jiosaavn` | `docker/jiosaavn.Dockerfile` | `/api/search/songs?query=test` |
| `OneTune-api` | `docker/api.Dockerfile` | `/health` |

**Docker context:** repository root (`.`).

**`OneTune-api` environment variables:**

| Key | Value |
|-----|--------|
| `NODE_ENV` | `production` |
| `MONGODB_URI` | Atlas URI from Part 1 |
| `JWT_SECRET` | `openssl rand -base64 48` |
| `EXTRACTOR_URL` | `https://OneTune-extractor.onrender.com` |
| `SPOTIFY_URL` | `https://OneTune-spotify.onrender.com` |
| `JIOSAAVN_URL` | `https://OneTune-jiosaavn.onrender.com` |
| `LOG_LEVEL` | `info` |

Use each service’s real `*.onrender.com` URL after the provider services are live.

## Part 4 — Verify the backend

Replace with your API URL (from Render → `OneTune-api` → **Settings** → URL):

```powershell
$api = "https://OneTune-api.onrender.com"

# Basic health (may be slow on first request if free tier was sleeping)
Invoke-RestMethod "$api/health"

# Dependency health — all should be "ok"
Invoke-RestMethod "$api/health/deps"
```

**Register a test user:**

```powershell
$body = @{ email = "you@example.com"; password = "your-secure-password"; displayName = "You" } | ConvertTo-Json
Invoke-RestMethod -Method Post -Uri "$api/v1/auth/register" -ContentType "application/json" -Body $body
```

If `health/deps` shows a provider failing, open that service’s **Logs** tab on Render and fix before shipping the app.

### Cold starts (free tier)

Render free services **sleep after ~15 min idle**. The first API request can take **30–60 seconds**. The mobile app should show loading/errors gracefully — upgrade to **Starter** plan on API (and providers) for always-on.

## Part 5 — Configure the mobile app

Your production API URL (example):

```
https://onetune-api.onrender.com
```

Use your actual Render hostname. **No trailing slash.**

### Step 1 — Set API URL for the build

Edit `apps/mobile/.env`:

```env
EXPO_PUBLIC_API_URL=https://onetune-api.onrender.com
EXPO_NO_METRO_WORKSPACE_ROOT=1
```

**HTTPS only** for release APKs — `app.config.js` disables Android cleartext when the URL uses `https://`.

---

## Part 6 — Build the mobile app (local)

### Step 1 — Standalone release APK (CLI)

```powershell
cd apps\mobile
bun run build:android:standalone
```

Output: `android\app\build\outputs\apk\release\app-release.apk`

First build: **15–30 minutes**.

### Step 2 — Install with ADB

```powershell
adb devices
adb install -r android\app\build\outputs\apk\release\app-release.apk
```

Enable **Install unknown apps** on the phone if prompted.

### Alternative — Android Studio

1. Run prebuild (if `android/` missing):

```powershell
cd apps\mobile
$env:EXPO_STANDALONE_BUILD="1"
$env:EXPO_NO_METRO_WORKSPACE_ROOT="1"
$env:EXPO_PUBLIC_API_URL="https://onetune-api.onrender.com"
npx expo prebuild --platform android --clean
```

2. Open `apps/mobile/android` in Android Studio.
3. Build variant: **release** → **Build → Build APK(s)**.
4. Install via ADB or drag onto emulator.

Full details: [DEPLOYMENT.md — Mobile APK](./DEPLOYMENT.md#3-mobile-apk-local-build).

## Part 7 — Install and test end-to-end

1. Install the **preview** or **production** APK on your phone.
2. Open the app → **Register** / **Login** (hits Render API).
3. **Search** → play a JioSaavn or YouTube track.
4. **Import** a playlist/album/song link (Library → Import music).
5. **Queue** — add tracks, verify skip / auto-advance.
6. **Favorites / History** — like a track, play a few, check Library.
7. **Downloads** — download a track, toggle airplane mode, play offline.

### Dev client vs standalone

| Build | When |
|-------|------|
| `npx expo run:android` + Metro | Daily coding on your PC |
| `bun run build:android:standalone` | Test against production API on a real phone |
| `bun run build:android:rebundle` | JS/API URL changed — faster than full prebuild |

## Security checklist

- [ ] Strong `JWT_SECRET` on Render (not the dev default)
- [ ] Atlas user has a strong password; `MONGODB_URI` only in Render env (never in git)
- [ ] `NODE_ENV=production` on API (disables `/v1/internal/*`)
- [ ] Mobile uses **HTTPS** API URL in preview/production
- [ ] Treat as **private / household** use — scraping has ToS/legal risk
- [ ] Rotate `JWT_SECRET` if leaked (invalidates all refresh tokens)

## Troubleshooting

| Symptom | Cause | Fix |
|---------|--------|-----|
| API build fails on `postinstall` | Mobile patch script in Docker | Fixed: `api.Dockerfile` uses `--ignore-scripts` |
| `health/deps` provider down | Provider still building / crashed | Render logs for that service; redeploy |
| API `502` / timeout | Cold start on free tier | Wait 60s, retry; or upgrade to Starter |
| Mobile “network error” | Wrong API URL or HTTP on HTTPS-only | Set `EXPO_PUBLIC_API_URL` to `https://…onrender.com`, rebuild |
| Atlas connection refused | Wrong URI or IP whitelist | Check `MONGODB_URI`, allow `0.0.0.0/0` |
| JioSaavn build OOM | Small Render instance | Use Starter plan; retry deploy |
| Auth works, search empty | Providers not wired | Check `EXTRACTOR_URL`, `JIOSAAVN_URL`, `SPOTIFY_URL` on API |
| Import fails Spotify album | Spotify service old image | Redeploy `OneTune-spotify` |
